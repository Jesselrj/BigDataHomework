# 语义代码复用检测实验报告

## 摘要

本实验围绕 POJ-104 语义代码检索任务构建并评估语义代码复用检测系统。POJ-104 中同一编程题下的不同 C/C++ 程序被视为语义等价代码，评测目标是在候选集合中检索出解决同一问题的程序。实验实现了 TF-IDF 词法检索基线、CodeBERT/GraphCodeBERT 代码对分类模型、UniXcoder 双塔检索模型，并在 UniXcoder 上提出 Label-aware 对比学习优化。

实验结果表明，预训练代码模型明显优于传统词法检索方法。TF-IDF 的 MAP@R 为 0.2169，本地复现的 UniXcoder baseline 达到 0.9098。本文提出的 Label-aware UniXcoder 进一步达到 0.9117，高于 UniXcoder 原文报告的 0.9052，也高于本地复现 baseline。该结果说明，在 POJ-104 这类按 problem_id 标注的检索任务中，避免 batch 内同类样本被误作为负例，可以改善 UniXcoder 的语义检索效果。


关键词：语义代码复用检测；代码克隆检测；POJ-104；UniXcoder；Label-aware Loss；对比学习

## 1. 实验目的

本实验的目标是构建一个可复现的 POJ-104 语义代码复用检测系统，用于识别不同程序之间是否具有相同语义。与 Type-1/Type-2/Type-3 克隆主要关注文本、变量名或局部结构相似不同，Type-4 代码克隆更强调程序语义一致，即两段代码即使实现方式、变量命名和控制结构存在差异，只要解决相同问题，也应被识别为语义等价。

本报告的主任务是代码语义检索：给定一个查询程序，从候选代码库中检索解决同一问题的程序。围绕该任务，实验比较 TF-IDF、UniXcoder baseline 和本文改进的 Label-aware UniXcoder，并以 MAP@R 作为主要评价指标。

CodeBERT、GraphCodeBERT 二分类实验作为扩展分析，用于辅助理解不同模型结构的表现，不作为本文主要结论。

## 2. 数据集与数据处理

实验使用 POJ-104 数据集。该数据集包含来自 104 个编程题的 C/C++ 程序，同一题目下的不同程序被视为语义等价代码。实验采用如下标注逻辑：

- 若 `problem_id` 相同，则构成正例，标签为 1。
- 若 `problem_id` 不同，则构成负例，标签为 0。

数据被处理为两种格式。第一种是检索格式，每条样本包含代码、样本 id、题目 id 和语言字段，用于语义检索任务。第二种是代码对格式，每条样本包含两段代码、两个题目 id、二分类标签和样本类型，用于 CodeBERT 与 GraphCodeBERT 的二分类训练和评估。

处理后的主要数据规模如下：

| 数据文件 | 样本数 |
|---|---:|
| `train.jsonl` | 32,500 |
| `validation.jsonl` | 8,500 |
| `test.jsonl` | 12,000 |
| `train_pairs.jsonl` | 26,000 |
| `validation_pairs.jsonl` | 6,800 |
| `test_pairs.jsonl` | 9,600 |
| `hard_negatives.jsonl` | 96,301 |

困难负例由三类样本组成：

| 困难负例类型 | 构造方式 | 数量 |
|---|---|---:|
| 随机负例 | 从不同 `problem_id` 中随机采样 | 32,500 |
| 词法困难负例 | 选择不同题目但 TF-IDF 相似度较高的代码对 | 31,301 |
| 长度/结构困难负例 | 选择不同题目但代码长度或 token 数相近的代码对 | 32,500 |

## 3. 方法设计

### 3.1 TF-IDF 词法检索基线

TF-IDF 基线首先对代码进行 token 化，将代码转化为词袋表示，再使用 TF-IDF 向量表示代码文本。检索阶段计算查询代码与候选代码之间的余弦相似度，并按相似度排序返回 Top-K 结果。该方法不使用深度模型，主要用于衡量词法相似性在语义代码检索中的基础效果。

### 3.2 CodeBERT 二分类模型

CodeBERT 二分类器使用 `microsoft/codebert-base` 作为编码器。输入形式为代码对拼接：

```text
code1 [SEP] code2
```

模型取编码后的分类表示，接一个线性分类头输出 clone / non-clone 概率。训练目标为交叉熵损失，评估指标包括 Accuracy、Precision、Recall、F1 和 AUC。

### 3.3 GraphCodeBERT 二分类与重排序模型

GraphCodeBERT 二分类器使用 `microsoft/graphcodebert-base` 作为编码器，输入和训练目标与 CodeBERT 相同。该模型在系统中承担两个角色：

1. 作为独立代码对二分类器。
2. 作为混合检索系统中的 Cross-Encoder 重排序器。

在重排序阶段，GraphCodeBERT 对 UniXcoder 检索得到的 Top-K 候选进行逐对打分，输出候选代码与查询代码的语义等价概率。

### 3.4 UniXcoder 双塔语义检索模型

UniXcoder 检索模型使用 `microsoft/unixcoder-base` 作为共享编码器，分别编码查询代码和候选代码：

```text
code -> UniXcoder -> vector
```

编码向量经过归一化后使用点积或余弦相似度计算匹配分数。训练时采用 in-batch negatives，即同一 batch 中其他样本作为负例，优化目标为 InfoNCE / Multiple Negatives Ranking Loss。该结构适合大规模候选池检索，因为所有候选代码可以预先编码并缓存。

### 3.5 Label-aware UniXcoder

原始 in-batch negative 训练默认一个 batch 中除配对正例外的其它样本都是负例。但在 POJ-104 中，同一 `problem_id` 下有大量语义等价程序；如果同一 problem 的多个样本同时出现在一个 batch 中，原始训练目标会把这些真实正例错误当作负例。

为减少这种假负例惩罚，本文在 UniXcoder 的 batch 内对比学习目标中加入 label-aware positive mask：同一 `problem_id` 的样本共同作为正例，不同 `problem_id` 的样本才作为负例。该改动不引入额外模型，只修改训练目标，属于直接作用在 UniXcoder 上的算法优化。

### 3.6 UniXcoder + GraphCodeBERT 两阶段混合方法

最终系统采用两阶段设计：

1. 第一阶段使用 UniXcoder 双塔模型进行快速召回，取得 Top-100 候选。
2. 第二阶段使用 GraphCodeBERT Cross-Encoder 对候选进行重排序。

最终得分定义为：

```text
final_score = alpha * retrieval_score + beta * cross_encoder_score
```

本实验默认使用 `alpha = 0.4`，`beta = 0.6`。该设计兼顾了双塔模型的检索效率和 Cross-Encoder 的精细匹配能力。

## 5. 评价指标

检索任务使用如下指标：

- MAP@R：主要指标。对每个 query，令 R 为候选池中相关样本数量，计算前 R 个结果上的平均精度，再对所有 query 取平均。
- Recall@1、Recall@5、Recall@10：衡量前 K 个结果中是否召回相关样本。
- MRR：Mean Reciprocal Rank，衡量第一个相关结果的排名位置。

二分类任务使用如下指标：

- Accuracy：分类准确率。
- Precision：预测为正例的样本中真正例比例。
- Recall：全部正例中被正确识别的比例。
- F1：Precision 与 Recall 的调和平均。
- AUC：基于预测概率的 ROC-AUC。

## 6. 实验结果

### 6.1 POJ-104 主结果

| 方法 | 任务 | MAP@R | Recall@1 | Recall@5 | Recall@10 | MRR | 说明 |
|---|---|---:|---:|---:|---:|---:|---|
| TF-IDF | Retrieval | 0.2169 | 0.8077 | 0.9140 | 0.9487 | 0.8561 | 词法基线 |
| UniXcoder 原文 | Retrieval | 0.9052 | - | - | - | - | 论文报告结果 |
| UniXcoder baseline | Retrieval | 0.9098 | 0.9976 | 0.9988 | 0.9993 | 0.9982 | 本地复现 |
| Label-aware UniXcoder | Retrieval | 0.9117 | 0.9982 | 0.9989 | 0.9992 | 0.9986 | 本文方法 |
| UniXcoder + GraphCodeBERT | Retrieval + Rerank | 0.9053 | 0.9979 | 0.9989 | 0.9992 | 0.9984 | 扩展实验 |
| Hybrid + Hard Negatives | Retrieval + Rerank | 0.8884 | 0.9978 | 0.9983 | 0.9986 | 0.9981 | 消融实验 |

从结果可以看出，TF-IDF 基线 MAP@R 仅为 0.2169，说明词法相似性无法稳定完成语义级代码检索。UniXcoder baseline 将 MAP@R 提升到 0.9098，证明预训练代码模型能够显著增强代码语义表示能力。

本文提出的 Label-aware UniXcoder 取得 MAP@R 0.9117，相比本地 UniXcoder baseline 提升 0.0019，也高于 UniXcoder 原文报告的 0.9052。提升幅度不大，但在 UniXcoder baseline 已经很强的情况下，仍说明假负例修正对 POJ-104 检索任务有正向作用。

### 6.2 扩展实验与消融

UniXcoder + GraphCodeBERT 重排序和 hard negative 实验作为扩展分析。重排序方法 MAP@R 为 0.9053，低于 Label-aware UniXcoder，说明当前融合权重更偏向提升首个正确结果，对 top-R 内整体相关样本排序并没有带来稳定增益。

| 设置 | MAP@R | Recall@1 | Recall@5 | Recall@10 | MRR |
|---|---:|---:|---:|---:|---:|
| UniXcoder + GraphCodeBERT | 0.9053 | 0.9979 | 0.9989 | 0.9992 | 0.9984 |
| Hybrid + Hard Negatives | 0.8884 | 0.9978 | 0.9983 | 0.9986 | 0.9981 |
| 差值 | -0.0169 | -0.0001 | -0.0006 | -0.0006 | -0.0003 |

加入 hard negative 后，混合方法 MAP@R 下降，说明当前困难负例采样和训练配置没有直接提升最终检索指标。不过，困难负例仍具有分析价值，可以帮助暴露模型在“词法相似但语义不同”的边界样本上的误判风险。


## 7. 错误分析

错误分析脚本对混合方法预测结果进行了样本级检查，覆盖了六类典型情况。

第一类是高词法相似但语义不同的代码对。例如 `problem_4` 与 `problem_52`、`problem_10` 与 `problem_15` 中存在 TF-IDF 分数接近 1.0 的代码对，但它们实际解决不同问题。这说明仅依赖词法相似性容易把结构模板相似、变量和循环形式相近的程序误判为语义等价。

第二类是低词法相似但语义相同的代码对。例如 `problem_100` 中样本 `test_9553` 与 `test_9811` 的 token Jaccard 仅为 0.1518，但二者属于同一问题。这类样本说明语义等价代码可以采用完全不同的变量组织、分支写法和实现风格，词法方法难以稳定识别。

第三类是超长代码被截断的情况。部分 `problem_100` 样本 token 数超过 2,000，而模型最大长度为 512。长代码截断可能导致模型无法看到完整逻辑，尤其是当关键判断或输出逻辑出现在后半部分时，会影响分类和重排序。

第四类是同一问题下不同算法策略的实现。例如同一题目中既可能出现通用算法实现，也可能出现针对输入范围或特定样例的特殊写法。这类样本在 token overlap 上很低，但语义标签相同，是 Type-4 克隆检测的核心难点。

第五类是 TF-IDF 可能成功但神经模型失败的样本。错误分析中发现 GraphCodeBERT 对若干高 token Jaccard 的正例给出较低分数，例如 `problem_86` 中某正例对 token Jaccard 为 0.7609，但 GraphCodeBERT 分数仅为 0.1865。这说明神经模型并非总是优于词法方法，在部分高词面重合正例上也可能出现 false negative。

第六类是神经模型成功但 TF-IDF 失败的样本。例如 `problem_81` 中多个 query 的混合模型 Top-1 结果正确，而 TF-IDF Top-1 检索到了 `problem_82`、`problem_84` 或 `problem_85` 的代码。这说明预训练语义模型能够在一定程度上摆脱表层 token 相似性的限制。

总体来看，错误分析证明两阶段混合方法在大多数语义检索场景下表现稳定，但仍受限于输入长度、训练负例构造方式和 Cross-Encoder 对部分边界样本的判断能力。

## 9. 结论

本实验完成了一个围绕 POJ-104 的语义代码复用检测系统，覆盖数据处理、模型训练、检索评估、结果汇总和错误分析。实验表明，预训练代码模型显著优于传统 TF-IDF 词法检索；本地复现的 UniXcoder baseline 在 POJ-104 上达到 MAP@R 0.9098。

本文的主要方法是 Label-aware UniXcoder：在 batch 内对比学习中，将同一 `problem_id` 的样本共同视为正例，避免语义等价代码被错误作为负例。该方法在 POJ-104 上取得 MAP@R 0.9117，高于 UniXcoder 原文报告的 0.9052，也高于本地复现 baseline 的 0.9098。由此可以说明，针对 POJ-104 标注结构修正 in-batch negative 的假负例问题，能够带来稳定的小幅提升。

扩展实验中，GraphCodeBERT 重排序和 hard negative 训练没有超过 Label-aware UniXcoder，说明在当前数据规模和参数设置下，简单重排序或困难负例增强并不一定提升 MAP@R。后续工作可以继续从更合理的负例采样、任务自适应 pooling、以及更细粒度的程序结构表示三个方向改进。
