# 语义代码复用检测实验报告

## 摘要

本实验围绕 POJ-104 语义代码检索任务构建并评估语义代码复用检测系统。POJ-104 中同一编程题下的不同 C/C++ 程序被视为语义等价代码，评测目标是在候选集合中检索出解决同一问题的程序。

实验实现了 TF-IDF 词法检索基线、UniXcoder 双塔检索 baseline、Label-aware UniXcoder，以及本文最终方法 UniXcoder + SupCon CE (k=2)。最终方法在 UniXcoder 上进行训练目标和 batch 采样优化，在测试集上取得 MAP@R 0.9254，高于本地复现 UniXcoder baseline 的 0.9098。

关键词：语义代码复用检测；代码克隆检测；POJ-104；UniXcoder；监督对比学习；P-K balanced batch

## 1. 实验目的

本实验的目标是构建一个可复现的 POJ-104 语义代码复用检测系统，用于识别不同程序之间是否具有相同语义。与 Type-1/Type-2/Type-3 克隆主要关注文本、变量名或局部结构相似不同，Type-4 代码克隆更强调程序语义一致，即两段代码即使实现方式、变量命名和控制结构存在差异，只要解决相同问题，也应被识别为语义等价。

本报告的主任务是代码语义检索：给定一个查询程序，从候选代码库中检索解决同一问题的程序。实验比较 TF-IDF、UniXcoder baseline、Label-aware UniXcoder 和 UniXcoder + SupCon CE (k=2)，并以 MAP@R 作为主要评价指标。

## 2. 数据集与数据处理

实验使用 POJ-104 数据集。该数据集包含来自 104 个编程题的 C/C++ 程序，同一题目下的不同程序被视为语义等价代码。实验采用如下标注逻辑：

- 若 `problem_id` 相同，则构成语义相关样本。
- 若 `problem_id` 不同，则构成语义不相关样本。

检索格式中，每条样本包含代码、样本 id、题目 id 和语言字段。

| 数据文件 | 样本数 |
|---|---:|
| `train.jsonl` | 32,500 |
| `validation.jsonl` | 8,500 |
| `test.jsonl` | 12,000 |

## 3. 方法设计

### 3.1 TF-IDF 词法检索基线

TF-IDF 基线首先对代码进行 token 化，将代码转化为词袋表示，再使用 TF-IDF 向量表示代码文本。检索阶段计算查询代码与候选代码之间的余弦相似度，并按相似度排序返回结果。该方法不使用深度模型，主要用于衡量词法相似性在语义代码检索中的基础效果。

### 3.2 UniXcoder 双塔语义检索模型

UniXcoder 检索模型使用 `microsoft/unixcoder-base` 作为共享编码器，分别编码查询代码和候选代码：

```text
code -> UniXcoder -> vector
```

编码向量经过归一化后使用点积或余弦相似度计算匹配分数。训练时采用 in-batch negatives，即同一 batch 中其他样本作为负例，优化目标为 InfoNCE / Multiple Negatives Ranking Loss。该结构适合大规模候选池检索，因为所有候选代码可以预先编码并缓存。

### 3.3 Label-aware UniXcoder

原始 in-batch negative 训练默认一个 batch 中除配对正例外的其它样本都是负例。但在 POJ-104 中，同一 `problem_id` 下有大量语义等价程序；如果同一 problem 的多个样本同时出现在一个 batch 中，原始训练目标会把这些真实正例错误当作负例。

为减少这种假负例惩罚，Label-aware UniXcoder 在 batch 内对比学习目标中加入 label-aware positive mask：同一 `problem_id` 的样本共同作为正例，不同 `problem_id` 的样本才作为负例。该改动不引入额外模型，只修改训练目标，属于直接作用在 UniXcoder 上的算法优化。

### 3.4 UniXcoder + SupCon CE (k=2)

本文最终方法进一步将训练数据从“成对正样本”改为“按 `problem_id` 组织的实例级监督对比学习”。具体做法是使用 P-K balanced batch：每个 batch 中抽取多个 `problem_id`，每个 `problem_id` 采样 `k=2` 个代码样本。同一 `problem_id` 的样本作为正例，不同 `problem_id` 的样本作为负例。

训练目标由两部分组成：

```text
loss = supervised_contrastive_loss + 0.2 * cross_entropy_loss
```

其中 supervised contrastive loss 直接优化 UniXcoder embedding 的类内聚合和类间分离；CE 辅助项只在训练阶段使用，用于增强不同题目类别之间的判别边界。最终检索仍使用 UniXcoder 双塔 embedding 计算相似度。

## 5. 评价指标

检索任务使用如下指标：

- MAP@R：主要指标。对每个 query，令 R 为候选池中相关样本数量，计算前 R 个结果上的平均精度，再对所有 query 取平均。
- Recall@1、Recall@5、Recall@10：衡量前 K 个结果中是否召回相关样本。
- MRR：Mean Reciprocal Rank，衡量第一个相关结果的排名位置。

## 6. 实验结果

| 方法 | 类别 | MAP@R | Recall@1 | Recall@5 | Recall@10 | MRR | 说明 |
|---|---|---:|---:|---:|---:|---:|---|
| TF-IDF | 词法基线 | 0.2169 | 0.8077 | 0.9140 | 0.9487 | 0.8561 | 传统词法检索 |
| UniXcoder 原文 | 论文结果 | 0.9052 | - | - | - | - | 原文报告指标 |
| UniXcoder baseline | 本地复现 | 0.9098 | 0.9976 | 0.9988 | 0.9992 | 0.9982 | 本地 baseline |
| Label-aware UniXcoder | 本文方法 | 0.9117 | 0.9982 | 0.9989 | 0.9992 | 0.9986 | 假负例修正 |
| UniXcoder + SupCon CE (k=2) | 本文方法 | 0.9254 | 0.9981 | 0.9991 | 0.9996 | 0.9986 | 最终主方法 |

从结果可以看出，TF-IDF 基线 MAP@R 仅为 0.2169，说明词法相似性无法稳定完成语义级代码检索。UniXcoder baseline 将 MAP@R 提升到 0.9098，证明预训练代码模型能够显著增强代码语义表示能力。

本文最终方法 UniXcoder + SupCon CE (k=2) 取得 MAP@R 0.9254，相比本地 UniXcoder baseline 提升 0.0156，也高于 UniXcoder 原文报告的 0.9052。该提升来自训练阶段的算法优化：P-K balanced batch 保证 batch 内存在同类正样本，监督对比学习减少假负例影响，CE 辅助项增强不同 `problem_id` 之间的判别边界。

## 7. 错误分析

错误分析聚焦语义检索中的典型困难样本：

第一类是高词法相似但语义不同的代码。不同题目的程序可能共享高度相似的循环、数组和输入输出模板，词法方法容易误判。

第二类是低词法相似但语义相同的代码。同一题目的代码可以使用完全不同的变量组织、分支结构和实现风格，词法方法难以稳定识别。

第三类是超长代码被截断的情况。部分样本 token 数超过 2,000，而模型最大长度为 512。长代码截断可能导致模型无法看到完整逻辑，尤其是当关键判断或输出逻辑出现在后半部分时，会影响表示质量。

第四类是同一问题下不同算法策略的实现。同一题目中既可能出现通用算法实现，也可能出现针对输入范围或特定样例的特殊写法。这类样本在 token overlap 上很低，但语义标签相同，是 Type-4 克隆检测的核心难点。

## 9. 结论

本实验完成了一个围绕 POJ-104 的语义代码复用检测系统，覆盖数据处理、模型训练、检索评估、结果汇总和错误分析。实验表明，预训练代码模型显著优于传统 TF-IDF 词法检索；本地复现的 UniXcoder baseline 在 POJ-104 上达到 MAP@R 0.9098。

本文的主要方法是 UniXcoder + SupCon CE (k=2)：在 batch 内按 `problem_id` 进行 P-K balanced sampling，每个题目采样 2 个代码样本；训练时使用监督对比学习拉近同题代码表示、拉远不同题代码表示，并加入轻量 CE 辅助约束。该方法在 POJ-104 上取得 MAP@R 0.9254，高于 UniXcoder 原文报告的 0.9052，也高于本地复现 baseline 的 0.9098。由此可以说明，针对 POJ-104 标注结构设计更合适的 batch 采样和监督对比目标，能够带来更明显的检索指标提升。
